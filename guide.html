Setting up a web page

This is a guide intended for people looking to set up a web page on the servers at Institut for Matematik, Aarhus University.
I assume you have access to your account on the imf-system, either using ssh (the ssh server is ssh1.imf.au.dk) or by direct access to a computer at the university.
Throughout the text, substitute your imf-login whenever you see username.

First, create a folder named 'public_html' in '/home/username/' (not on the desktop).
In this folder, put a file named 'index.html'. This is the starting page when people go to http://home.imf.au.dk/username.
Any additional pages, images and files also go in the 'public_html' folder or a subfolder.

Start a terminal and type the following:

cd /home/
chmod o+x username
cd username
mkdir -p public_html
chmod o+x public_html
cd public_html
chmod o+r index.html
chmod o+r any-additional-files

These commands allow other people to access your web page. Without them, you will get a page saying you do not have permission to access the link when visiting http://home.imf.au.dk/username.

The page should now be ready to go. If you add other pages or files, remember to run:

cd /home/username/public_html
chmod o+r name-of-file

in a terminal, to allow other people access to them.

If you need/want to use utf-8 encoding (or some other type of encoding), you need to tell this to the server. This is done by adding a file calles .htaccess in the public_html folder with the single line

AddCharset utf-8 .html

and making it readable as before. This will tell the server that your .html files should be encoded with utf-8. Here is a shell-script to get you started. Download it, open a terminal, navigate to the download directory (often the desktop) and run:

sh imf-setup.sh username

This will create 'public_html', an empty file named 'index.html' and set all permissions, including using utf-8 for .html files.
As far as possible, the script should only do what is necessary, so if you already have a 'public_html' folder or an 'index.html' page it will only change permissions.
